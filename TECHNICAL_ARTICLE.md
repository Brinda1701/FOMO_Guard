## 📖 一、产品概述

**FOMOGuard（追涨护盾）** 是一款创新的投资心理辅助工具，融合了 **行为金融学理论** 与 **AI 多智能体协作技术**，帮助投资者在市场情绪波动中保持理性，避免追涨杀跌的情绪化交易。

### 产品定位
- **目标用户**：意识到自己有情绪化交易倾向但难以自控的投资者
- **核心价值**：不是"帮你赚钱"，而是"帮你少亏钱"
- **差异化**：不是"又一个股票分析工具"，而是"投资心理教练"

## ✨ 二、核心功能亮点

### 1. 🧠 冷静期机制（核心创新）

当用户做出高风险决策（如情绪高涨时追涨）时，系统强制启动 **100 秒冷静期**：

- **环形倒计时**：可视化进度，增强时间感知
- **呼吸引导**：4 秒吸气/呼气循环，帮助平复情绪
- **心理学问答**：正确回答可提前解锁，寓教于乐
- **阶段提示**：深呼吸→认知反思→理性评估，科学设计

```
用户场景：
"这只股票涨得太好了，我要马上买入！"
    ↓
FOMOGuard 检测到情绪分数 76（贪婪）
    ↓
🚨 启动冷静期：请等待 100 秒，或回答心理学问题
    ↓
100 秒后... "我刚才太冲动了，再看看基本面吧"
```

### 2. 🤖 Multi-Agent 智能分析

三个专业 Agent 协作分析，提供多维度评估：

| Agent | 职责 | 分析内容 |
|-------|------|---------|
| **😊 情绪分析 Agent** | 市场舆论 | 社交媒体、新闻情绪、FOMO 程度 |
| **📊 技术分析 Agent** | 技术指标 | 趋势、动量、超买超卖 |
| **🧠 心理诊断 Agent** | 认知偏误 | 锚定效应、确认偏误、羊群效应 |

**技术特色**：
- 支持 **SSE 流式输出**，实时展示分析进度
- 自动融合三 Agent 结果，生成一致性判断
- 降级机制：AI 不可用时自动切换 Mock 模式

### 3. 📰 新闻 URL 爬取分析

支持主流财经媒体链接的情绪分析：

- ✅ 雪球
- ✅ 东方财富/股吧
- ✅ 新浪财经
- ✅ 华尔街见闻

**技术实现**：
```javascript
// 后端爬取 → 提取标题/正文 → AI 情感分析 → 输出情绪分数
POST /api/scrape-url
{
  "url": "https://xueqiu.com/..."
}
→ { "score": 72, "sentiment": "正面", "summary": "..." }
```

### 4. 📊 批量分析 + CSV 导出

一次性分析最多 10 个公司，支持导出 CSV：

| 排名 | 公司 | 情绪分数 | 情绪状态 | 行业 | 操作 |
|------|------|---------|---------|------|------|
| 1 | 茅台 | 76 | 贪婪 | 消费 | 分析 |
| 2 | 比亚迪 | 70 | 贪婪 | 新能源 | 分析 |
| 3 | 特斯拉 | 46 | 中性 | 科技 | 分析 |

### 5. 📔 交易决策日记

记录每次交易决策，形成个人投资成长档案：

- **标的名称**：股票/基金名称
- **决策类型**：买入/卖出/观望
- **情绪评分**：0-100 分
- **决策心理**：（可选）记录当时想法

**价值**：定期回顾，识别自己的情绪化交易模式

### 6. 🎨 情绪粒子背景

根据市场情绪动态变化的视觉效果：

- 🟢 **贪婪**（>60 分）：绿色粒子上升
- 🟡 **中性**（40-60 分）：黄色粒子漂浮
- 🔴 **恐惧**（<40 分）：红色粒子下沉

## 💡 三、创意来源

### 行为金融学理论基础

FOMOGuard 的设计灵感来源于经典行为金融学理论：

| 理论 | 提出者 | 应用 |
|------|--------|------|
| **损失厌恶** | Kahneman & Tversky (1979) | 解释为何散户"割肉"痛苦 |
| **锚定效应** | Tversky & Kahneman (1974) | 解释为何散户"抄底"执念 |
| **羊群效应** | Shiller (1984) | 解释为何散户"追涨杀跌" |
| **处置效应** | Shefrin & Statman (1985) | 解释为何"过早止盈，过久持亏" |

### 真实痛点

> "我知道追高不好，但就是控制不住自己"

这是无数散户投资者的真实写照。FOMOGuard 的目标不是"预测市场"，而是**帮助用户对抗人性弱点**。

### 设计原则

1. **干预而非预测**：不预测股价涨跌，只评估情绪风险
2. **教育而非说教**：通过问答和提示，让用户自己领悟
3. **量化而非模糊**：将抽象的"感觉"变成具体数字

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | Vanilla JS (ES Modules) | 零框架依赖，轻量快速 |
| **UI** | CSS3 + Chart.js | 响应式设计 + 数据可视化 |
| **后端** | Vercel Serverless | 无服务器部署，自动扩展 |
| **AI** | ModelScope 魔搭 | 国内可访问的大模型 API |
| **存储** | localStorage | 本地持久化，保护隐私 |

### 核心 API

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/analyze` | POST | AI 情绪分析 |
| `/api/orchestrator` | POST | Multi-Agent 编排（支持 SSE） |
| `/api/scrape-url` | POST | URL 内容爬取 |
| `/api/batch-analyze` | POST | 批量分析 |
