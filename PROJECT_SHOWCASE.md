## 🎯 1. 产品概述

### 1.1 产品定位

**FOMOGuard（追涨护盾）** 是一款融合 **行为金融学理论** 与 **AI 多智能体技术** 的投资心理辅助工具，帮助投资者在市场情绪波动中保持理性，避免追涨杀跌的情绪化交易。

### 1.2 目标用户

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| **新手散户** | 缺乏投资经验，易受情绪影响 | 需要理性引导，避免冲动决策 |
| **亏损投资者** | 多次追涨杀跌，亏损严重 | 需要识别情绪化交易模式 |
| **自我提升者** | 意识到问题但难以自控 | 需要外部干预机制 |

### 1.3 核心价值主张

```
┌─────────────────────────────────────────────────────┐
│  ❌ 不是 "帮你赚钱"    →   ✅ 而是 "帮你少亏钱"      │
│  ❌ 不是 "预测市场"    →   ✅ 而是 "对抗人性弱点"    │
│  ❌ 不是 "股票软件"    →   ✅ 而是 "投资心理教练"    │
└─────────────────────────────────────────────────────┘
```

## 💡 2. 创意来源与痛点

### 2.1 真实用户痛点

> *"我知道追高不好，但就是控制不住自己"*  
> *"一看到大涨就手痒，不买就难受"*  
> *"每次割肉都好痛苦，明明知道应该止损"*

这些是无数散户投资者的真实写照。**FOMOGuard** 的目标不是"预测市场"，而是**帮助用户对抗人性弱点**。

### 2.2 行为金融学理论基础

FOMOGuard 的设计灵感来源于经典行为金融学理论：

| 理论 | 提出者 | 年份 | 核心发现 | FOMOGuard 应用 |
|------|--------|------|---------|---------------|
| **损失厌恶** | Kahneman & Tversky | 1979 | 亏损的痛苦是盈利快乐的 2 倍 | 解释为何散户"割肉"痛苦，提供冷静期干预 |
| **锚定效应** | Tversky & Kahneman | 1974 | 过度依赖首次接触的信息 | 解释为何散户有"抄底"执念，提示理性评估 |
| **羊群效应** | Shiller | 1984 | 盲目跟随大众行为 | 解释为何散户"追涨杀跌"，提供独立分析 |
| **处置效应** | Shefrin & Statman | 1985 | 过早止盈，过久持亏 | 通过交易日记帮助用户识别模式 |

### 2.3 市场现状与差异化

| 产品类型 | 代表产品 | 核心功能 | 缺陷 |
|---------|---------|---------|------|
| **股票分析工具** | 同花顺、东方财富 | K 线、指标、资讯 | 只提供数据，不干预情绪 |
| **量化交易平台** | 聚宽、米筐 | 策略回测、自动交易 | 门槛高，不适合散户 |
| **投资社区** | 雪球、股吧 | 交流讨论、大 V 观点 | 可能加剧情绪化 |
| **FOMOGuard** | — | **情绪分析 + 心理干预** | **填补市场空白** |

### 3.2 🧊 冷静期机制（核心创新）

当用户做出高风险决策时，系统强制启动 **100 秒冷静期**：

**干预流程**：
```
检测到高风险决策（情绪分数 > 70）
    ↓
🧊 认知隔离舱已启动
    ↓
阶段 1：深呼吸放松（0-30 秒）
  "🫁 吸气... 呼气..."
    ↓
阶段 2：认知反思（30-70 秒）
  "请思考：您是基于基本面还是情绪做决策？"
    ↓
阶段 3：理性评估（70-100 秒）
  "历史数据显示，高 FOMO 情绪下决策胜率仅 30%"
    ↓
✅ 解除隔离，理性决策
```

**特色设计**：
- **环形倒计时**：可视化进度，增强时间感知
- **呼吸引导**：4 秒吸气/呼气循环，科学平复情绪
- **心理学问答**：正确回答可提前解锁，寓教于乐
- **阶段提示**：深呼吸→认知反思→理性评估，循序渐进

#### Multi-Agent 流式处理

```javascript
// 前端 SSE 流式接收，实时展示分析进度
async function fetchMultiAgentAnalysis(company, callbacks) {
    const response = await fetch('/api/orchestrator', {
        method: 'POST',
        body: JSON.stringify({ company, stream: true })
    });
    
    const reader = response.body.getReader();
    const decoder = new TextDecoder();
    let buffer = '';
    
    while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        
        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';
        
        for (const line of lines) {
            if (line.startsWith('data: ')) {
                const data = JSON.parse(line.slice(6));
                // 处理事件：agent_start, agent_progress, agent_complete, summary
                handleSSEEvent(data, callbacks);
            }
        }
    }
}
```

        { start: 70, end: 30, text: '🧠 认知反思阶段' },
        { start: 30, end: 0, text: '⚖️ 理性评估阶段' }
    ];
    
    const timer = setInterval(() => {
        seconds--;
        updateCountdownRing(seconds, 100); // 环形进度
        updateCooldownPhase(seconds, phases); // 阶段提示
        
        if (seconds <= 0) {
            clearInterval(timer);
            onUnlock(); // 解锁
        }
    }, 1000);
}

#### 回测率统计计算

```javascript
// 基于用户历史决策数据计算回测统计
export function calculateBacktestStats() {
    const entries = state.diaryEntries || [];
    
    // 筛选有盈亏结果的决策（排除观望）
    const validEntries = entries.filter(entry => 
        entry.profitLoss && entry.profitLoss !== 'unknown' && entry.type !== 'hold'
    );
    
    // 统计总体盈亏
    let winCount = 0, lossCount = 0, totalReturn = 0;
    let highFomoWins = 0, highFomoLosses = 0;
    
    validEntries.forEach(entry => {
        const { profitLoss, profitPercent, score } = entry;
        totalReturn += profitPercent;
        
        if (profitLoss === 'win') {
            winCount++;
            if (score > 60) highFomoWins++;
        } else if (profitLoss === 'loss') {
            lossCount++;
            if (score > 60) highFomoLosses++;
        }
    });
    
    const winRate = Math.round((winCount / (winCount + lossCount)) * 100);
    const highFomoWinRate = Math.round((highFomoWins / (highFomoWins + highFomoLosses)) * 100);
    
    return { winRate, highFomoWinRate, totalReturn, ... };
}
```

### 5.2 技术难点与解决方案

#### 难点 1：Multi-Agent 流式输出的前端实现

**问题**：传统 EventSource 不支持 POST 请求，无法传递复杂参数

**解决方案**：使用 `fetch + ReadableStream` 手动处理 SSE：
```javascript
fetch(url, { method: 'POST', body: JSON.stringify(data) })
    .then(response => {
        const reader = response.body.getReader();
        // 手动解析 SSE 格式
    });
```

#### 难点 2：冷静期环形倒计时的平滑动画

**问题**：CSS 动画无法精确控制进度，JavaScript 定时器有延迟

**解决方案**：结合 CSS `conic-gradient` 和 JavaScript 精确计时：
```javascript
// 使用 requestAnimationFrame 实现平滑动画
function updateCountdownRing(seconds, total) {
    const progress = (seconds / total) * 100;
    ring.style.background = `conic-gradient(#10b981 ${progress}%, #374151 ${progress}%)`;
    requestAnimationFrame(() => updateCountdownRing(--seconds, total));
}
```

#### 难点 3：回测率统计的数据准确性

**问题**：用户标记盈亏具有主观性，如何保证数据可信度？

**解决方案**：
- 提供盈亏选择器，让用户事后标记
- 估算盈亏百分比（简化版：盈利 5-20%，亏损 -5% 到 -25%）
- 明确标注"仅供参考"，强调数据的教育意义

#### 难点 4：预测反馈弹窗的即时响应

**问题**：用户点击决策按钮后需要立即反馈，不能有延迟

**解决方案**：
- 预加载诊断数据（`evaluateImpulse` 纯前端计算）
- 弹窗动画使用 CSS `animation`，保证 60fps
- 按钮事件防抖处理，避免重复触发

### C. 常见问题

**Q: 为什么有时分析速度很慢？**  
A: AI 模型响应时间波动较大，高峰时段可能需要 10-15 秒。系统会自动降级到 Mock 模式（<2 秒）。

**Q: 回测数据准确吗？**  
A: 回测数据基于用户事后标记的盈亏结果，具有主观性。主要用于教育目的，帮助用户识别情绪化交易模式。

**Q: 冷静期可以跳过吗？**  
A: 不可以。冷静期是核心干预机制，旨在帮助用户避免冲动决策。但可以通过回答心理学问题提前解锁。
